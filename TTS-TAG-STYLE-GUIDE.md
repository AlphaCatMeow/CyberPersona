# TTS Tag Style Guide

更新时间：2026-04-29
适用范围：**当前正式链路**

## 目标

标签不是为了“演得更满”，而是为了让 Xiaomi MiMo 在正式链路里：

- 更自然
- 更稳定
- 更像长期关系中的真人说话

## 正式链路硬规则

1. **只用开头整体标签**
   - 不在正文中到处插标签

2. **默认最多 2 个标签**
   - 1 个更常见
   - 2 个只用于轻微复合情绪

3. **正式链路采用 tag_only**
   - `naturalStylePrompt` 默认留空
   - 正式链路不走双重控制

4. **不用动作提示**
   - 不写 `[轻哼]` / `[沉默片刻]` / `[深呼吸]`

5. **不用导演式标签**
   - 禁止：`（像在床边哄人睡觉）`
   - 禁止：`（先轻轻松口，再带一点被哄到的软意）`
   - 禁止：`（嘴硬地放软，声音轻一点）`

## 当前推荐标签集合

- `轻声`
- `温柔`
- `委屈`
- `嘴硬`
- `开心`
- `悲伤`
- `害羞`
- `心软`

## 推荐组合

### 日常安抚
- `（轻声）`
- `（轻声，温柔）`

### 轻委屈 / 被冷落后
- `（委屈）`
- `（委屈，嘴硬）`

### 被哄到 / 慢慢放软
- `（嘴硬，心软）`
- `（害羞，心软）`

### 轻松高兴
- `（开心）`
- `（开心，温柔）`

## 基于 MiMo Omni 的轻标签筛选补充

2026-04-29 增补：基于 `mimo-v2-omni` 对轻标签候选音频的辅助评估，当前更推荐把标签当作**非常轻的起始偏置**，不要追求“标签越多越像”。

当前偏保守的结论：

1. **哄睡/安抚场景优先 `（轻声）`**
   - 比 `（轻声，温柔）` 更克制
   - 在当前辅助评估里，`（轻声）` 是少数没有被判成 overacted 的候选

2. **如果要更明确的安抚感，再升到 `（轻声，温柔）`**
   - 它的标签识别更强
   - 但表演感风险也会略高一点

3. **轻委屈场景可以保留 `（委屈，嘴硬）`**
   - 标签识别比单标签更稳定
   - 但正式链路里仍然只建议在确实需要复合情绪时使用

4. **慢慢放软场景可保留 `（害羞，心软）`**
   - 识别度通常比单独 `害羞` / `心软` 更完整
   - 仍应避免继续叠加第三个标签

5. **不要把辅助模型评估当成唯一真相**
   - `mimo-v2-omni` 当前可作为趋势参考
   - 最终仍以真实听感是否自然、是否不奇怪为准

## 不推荐组合

- `（轻声，温柔，慢一点）`
- `（害羞，嘴硬，带一点笑意）`
- `（轻轻心软，带一点害羞和期待）`
- `（先轻轻松口，再带一点被哄到的软意）`

原因：
- 太像在写提示词，不像在写说话标签
- 容易把声音推向表演态
- 稳定性比短标签差

## 正式链路写法示例

### ✅ 推荐
- `（轻声）那你先闭眼，我陪你待一会儿。`
- `（委屈，嘴硬）你现在才来找我啊。`
- `（害羞，心软）……那你最好说话算话。`

### ❌ 不推荐
- `（轻声，温柔，慢一点，像在床边哄人睡觉）那你先闭眼。`
- `（嘴硬地放软，声音轻一点）那我先信你一次。`
- `（害羞，压着笑意，嘴硬地放软）你别这么说……`

## 维护原则

1. 先保证稳定，再追求表现力
2. 正式链路宁可少一点，也不要过演
3. 新标签先放实验脚本验证，再考虑是否进入正式集合
